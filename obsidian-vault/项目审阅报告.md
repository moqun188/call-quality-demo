## 一、项目概览

### 1.1 核心定位
**基于 Node.js + 小米 MiMo API 的客服录音质检系统 MVP Demo**

### 1.2 技术架构
- **后端**: Node.js + Express
- **AI 引擎**: 小米 MiMo (mimo-v2.5-asr / mimo-v2.5-pro)
- **音频处理**: ffmpeg + fluent-ffmpeg
- **前端**: 原生 HTML/CSS/JS（无框架）
- **知识库**: Obsidian Markdown 格式

### 1.3 核心流程
```
上传音频 → 格式统一转码 → ASR语音识别 → 说话人分离 → 情绪分析 → 通话总结 → 质检评分
                                                              ↓
                                                      导出质检报告（Markdown/Excel/JSON）
```

### 1.4 模块清单

| 模块 | 文件 | 职责 | 优先级 |
|------|------|------|--------|
| 编排器 | pipeline/index.js | 串联各步骤，回调推送进度 | 🔴 |
| 预处理 | pipeline/preprocessor.js | 格式检测、统一转码为 MP3 | 🟠 |
| 音频转换 | pipeline/audioConverter.js | ffmpeg 实际转码实现 | 🟠 |
| ASR | pipeline/asr.js | 小米 ASR 转写 + 多策略解析 | 🔴 |

| 质检评分 | pipeline/qualityAnalyzer.js | 规则引擎评分（4维度） | 🔴 |
| 统计管理 | pipeline/statsManager.js | 历史记录持久化（JSON文件） | 🟠 |
| 日志 | pipeline/logger.js | 控制台+文件日志 | 🟡 |
| Obsidian 导出 | pipeline/obsidianExporter.js | 生成 Markdown 笔记 | 🟢 |

## 四、存在的问题与改进建议 🔧

### 🔴 P0 - 紧急修复

#### 问题 1: API 调用重复 + 成本浪费
**严重程度**: 高
**问题描述**:
当前流程中，4 个 AI 模块（ASR / 说话人分离 / 情绪分析 / 总结）各自独立调用小米 API，每次都需要：
- 读取整个音频文件 → 转 Base64
- HTTP 请求发送到小米 API
- 等待模型处理（10-30 秒）

**影响**:
- Credits 消耗严重：每次质检 ≈ 3 次音频 + 1 次纯文本 API 调用
- 速度慢：串行等待时间叠加，一段 3 分钟音频可能需要 1-2 分钟
- 冗余 IO：同一音频文件被 3 个模块重复读取和编码

**改进建议**:
合并 ASR + 说话人分离 + 情绪识别为单请求，使用 mimo-v2.5-pro 一次性产出所有结果：
```javascript
// 优化后的 prompt
const prompt = `请分析这段客服录音，返回以下 JSON：
{
  "transcript": [
    {"speaker": "agent"|"customer", "start": 秒, "end": 秒, "text": "文本", "emotion": "平静|愤怒|..."}
  ],
  "summary": {"callPurpose": "...", "keyPoints": [...], ...}
}`;
```
**收益**: 节省 75% Credits，速度提升 3-4 倍

#### 问题 2: statsManager.avgDimensions 计算 Bug
**严重程度**: 高
**位置**: pipeline/statsManager.js 第 97 行
**问题代码**:
```javascript
avgDimensions[k] = Math.round(
  inspections.reduce((s, r) => s + (r.dimensions[k] || 0), 0)  
  // 问题：r.dimensions[k] 是对象 { score, reason }，不是数字！
  / total * 10
) / 10;
```
**修复方案**:
```javascript
avgDimensions[k] = Math.round(
  inspections.reduce((s, r) => s + (r.dimensions[k]?.score || 0), 0)  
  / total * 10
) / 10;
```
