# 📌 CallQ Sprint 4 任务看板（三视角优化版）

> 最后更新：2026-06-19 08:35  
> 分配依据：马斯克（成本/架构） + 乔布斯（体验/聚焦） + Karpathy（抽象层/可靠性）  
> 原则：先减法后乘法，先能用再好用

---

## 🧭 三视角共识

三人对这个项目的核心判断出奇一致：

| 维度 | 马斯克 | 乔布斯 | Karpathy | 共识 |
|------|--------|--------|----------|------|
| LLM 调用 | 大炮打蚊子 | 不重要的功能砍掉 | 错误的抽象层 | **5 次→1 次，只留 ASR** |
| 产品形态 | 专用 ASR + 规则引擎 | 早上打开看 5 个红色标记 | Iron Man 套装模式 | **Dashboard + 问题高亮** |
| 质量保障 | 白痴指数验证 | 要么最好要么垃圾 | 没有测试集就是赌博 | **必须有 ground truth** |
| 数据飞轮 | 垂直整合 | 用户不知道自己要什么 | 先看数据再训练 | **质检反馈 → 规则迭代** |

## 🔴 Sprint 4 — 架构瘦身（6/19 ~ 6/22）

### 马斯克刀：砍成本

| # | 任务 | 负责人 | 预估 | 说明 |
|---|------|--------|------|------|
| M-001 | **砍掉独立情绪分析模块** | moqun188 | 2h | 删除 `emotion.js` 的独立 API 调用，情绪标签由多模态调用一次产出（或直接由规则引擎从文本推断） |
| M-002 | **砍掉独立说话人分离模块** | moqun188 | 2h | 删除 `diarization.js` 的独立 API 调用。说话人分离只在多模态调用中附带完成，不再单独调用 |
| M-003 | **合并 summarizer 到规则引擎** | moqun188 | 3h | 总结生成改为模板化：从质检结果拼装结构化总结，不再每次调用 LLM。只在用户明确要求"AI 总结"时才调用 |
| M-004 | **API 调用从 5 次→1 次** | moqun188 | 1h | pipeline 只保留 1 次多模态调用（ASR+说话人+情绪），其余全部本地规则引擎 |

**目标**：单次质检 API 成本降低 80%，延迟从 3-8 秒降到 1-3 秒。

### 乔布斯刀：砍功能

| # | 任务 | 负责人 | 预估 | 说明 |
|---|------|--------|------|------|
| J-001 | **砍掉 Obsidian 导出** | sxbwork | 1h | 从 server.js 移除 `/api/export/obsidian` 端点和 `obsidianExporter.js`。开发者自我满足，不是产品功能 |
| J-002 | **砍掉自我进化引擎** | sxbwork | 1h | 移除 `selfEvolution.js` 和 `evolution.html`。产品还没做好就做"自我进化"，是功能膨胀 |
| J-003 | **砍掉 Token 用量统计页** | sxbwork | 0.5h | 移除 `token-usage.html`。开发者工具不是用户功能 |
| J-004 | **砍掉 What's New 页面** | sxbwork | 0.5h | 移除 `whats-new.html`。没人需要看更新日志 |
| J-005 | **砍掉 ASR 测试页** | sxbwork | 0.5h | 移除 `asr-test.html`。开发者调试工具不是产品 |
| J-006 | **只保留 Excel 导出** | sxbwork | 1h | JSON 导出砍掉，只留 Excel（用户友好） |

**目标**：前端页面从 7 个减到 3 个（首页、Dashboard、批量质检），代码量减少 40%。

### Karpathy 刀：加测试

| # | 任务 | 负责人 | 预估 | 说明 |
|---|------|--------|------|------|
| K-001 | **构建质检测试集** | 乔布斯 | 4h | 收集/构造 20 通测试录音（10 通好、10 通差），人工标注期望的质检结果。没有 ground truth 就是在赌博 |
| K-002 | **质检准确率 benchmark** | moqun188 | 3h | 用测试集跑 qualityAnalyzer，输出 precision/recall/F1。现在你不知道系统准不准 |
| K-003 | **ASR 准确率测试** | moqun188 | 2h | 用测试集对比 ASR 输出与人工转写，计算字错率（CER）。目标：< 10% |
| K-004 | **前端回归测试** | sxbwork | 2h | 核心流程（上传→结果→导出）的 E2E 测试，确保砍功能后不崩 |

**目标**：建立可量化的质量基线，告别"看起来合理"。
