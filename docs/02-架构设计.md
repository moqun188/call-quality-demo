## 📦 核心模块说明

### Pipeline 模块（pipeline/index.js）
**职责**：编排整个质检流程，协调各子模块

**输入/输出**：
```
输入: 音频文件路径 + 配置参数
输出: { utterances（带情绪/说话人标注的对话
       emotion（整体情绪摘要）
       quality（4 维度评分）
       summary（结构化总结）
       步骤状态对象 （前端呈现）
```

**关键逻辑**：
1. 调用 preprocessor → 统一格式
2. 调用 ASR → 语音转写
3. 调用 diarization → 说话人分离
4. 调用 emotion → 情绪分析
5. 调用 summarizer → 通话总结
6. 调用 qualityAnalyzer → 质检评分

---

### ASR 模块（pipeline/asr.js）
**职责**：将音频转写为文本，支持模拟和真实两种模式

**模式切换**：
- `ENABLE_REAL_ASR=true → 调用小米 MiMo API
- `ENABLE_REAL_ASR=false` → 返回模拟数据

**返回格式**：
```json
{
  "model": "mimo-v2.5-asr",
  "duration": 42.5,
  "segments": [
    {
      "start": 0.0,
      "end": 3.2,
      "text": "您好，请问有什么可以帮您？",
      "confidence": 0.93
    }
  ]
}
```

---

### 说话人分离（pipeline/diarization.js）
**职责**：为每一句对话标注说话人（agent / customer）

**关键方法**：
- `diarize(audioPath)` → 返回说话人分段
- `align(asrSegments, diarSegments)` → 将 ASR 结果与说话人对齐

---

### 情绪分析（pipeline/emotion.js）
**职责**：逐句分析情绪，并给出整体对话情绪趋势

**8 种情绪标签**：
```
平静、愉悦、焦急、愤怒、不满、困惑、冷漠、惊讶
```

**3 维度量**：
```
valence  (情感极性: 消极 ↔ 积极)
arousal (激进度: 平静 ↔ 激动)
dominance (主导度: 顺从 ↔ 主导)
```

---

### 通话总结（pipeline/summarizer.js）
**职责**：使用 LLM 生成结构化通话总结

**输出字段**：
```json
{
  "callPurpose": "客户咨询退货流程",
  "keyPoints": ["要点1", "要点2"],
  "customerRequest": "客户主要诉求...",
  "resolutionStatus": "已解决 / 待跟进 / 未解决",
  "actionItems": ["后续行动项"],
  "qualityIssues": ["质量问题"],
  "overallAssessment": "整体评价",
  "highlights": "客服亮点描述",
  "improvementSuggestions": ["改进建议"],
  "generatedBy": "mimo-v2.5-pro"
}
```

---

### 质检评分（pipeline/qualityAnalyzer.js）
**职责**：基于规则引擎的 4 维度质检评分

**4 个评分维度**：
| 维度 | 权重 | 检查内容 |
|------|------|---------|
| 话术合规 | 30% | 开场白、结束语、礼貌用语、禁语 |
| 业务知识 | 25% | 信息准确性、问题解答完整性 |
| 流程完整 | 25% | 关键环节缺失检查 |
| 沟通技巧 | 20% | 共情表达、引导能力、情绪处理 |

## 📚 模块边界规范

### ✅ 模块之间应该
- 通过明确的函数接口交互
- 输入/输出格式在文档中定义
- 错误向上抛出，由调用方处理
- 日志输出结构化信息

### ❌ 模块之间不应该
- 直接修改其他模块的数据结构
- 依赖模块内部实现细节
- 共享全局变量（除日志、配置外）
- 循环依赖

---

## 🔌 API 调用策略

### 当前问题
- 4 次独立的 MiMo API 调用
- 每次都要重新编码和等待
- Credits 消耗高、速度慢

### 优化方向（P0）
合并为单次多模态调用，prompt 设计：
```
输入: 音频文件 + 结构化输出格式说明
输出: 一次返回 ASR + 说话人 + 情绪 + 总结
```

### 降级策略
- API 失败 → 自动降级到模拟模式
- 记录失败原因到日志
- 为人工修复提供入口
