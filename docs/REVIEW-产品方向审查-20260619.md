## 二、产品方向建议

### 核心定位（建议）

> **面向中小企业的 AI 客服质检 SaaS 工具**
> - 上传录音 → 自动质检 → 生成报告
> - 低门槛、快部署、按量付费

### 为什么不走"大企业定制"路线？

1. 团队 2 人开发，资源有限
2. 大企业需要私有化部署、CRM 对接、复杂权限——这些都是 P3 的事
3. 中小企业市场更缺工具，付费意愿更直接

### 优先级重排建议

| 优先级 | 事项 | 原因 |
|--------|------|------|
| **P0** | 竞品分析完成 | 没有竞品分析，后面所有功能优先级都是拍脑袋 |
| **P0** | 质检规则可配置化 | 硬编码规则 = 只能服务一个行业 |
| **P1** | 用户场景梳理 + 痛点验证 | 找 3-5 个真实客服团队聊需求 |
| **P1** | 报告导出增强（PDF） | Excel/JSON 不够，客户要 PDF 报告 |
| **P2** | 多租户 + JWT 认证 | SaaS 基础能力 |
| **P2** | 前端 Vue 3 重构 | 提升开发效率和用户体验 |

## 三、需求 Gap 分析

### 当前已实现 vs 市场需求

| 能力 | 当前状态 | 市场需求 | Gap |
|------|----------|----------|-----|
| ASR 转写 | ✅ MiMo API | 准确率 95%+ | 需要实测准确率数据 |
| 说话人分离 | ✅ 基本可用 | 多人会议、方言场景 | 仅支持 2 人对话 |
| 情绪分析 | ✅ 8 种标签 | 精细化情绪（如"失望"vs"愤怒"） | 标签粒度不够 |
| 质检评分 | ✅ 4 维度 | 可自定义维度和权重 | 硬编码，无法配置 |
| 报告导出 | ✅ Excel/JSON/Obsidian | PDF 报告、邮件发送 | 缺 PDF |
| 批量质检 | ❌ 无 | 批量上传、队列处理 | 完全缺失 |
| 历史对比 | ❌ 无 | 趋势分析、同比环比 | 仅基础统计 |
| 实时质检 | ❌ 无 | 通话中实时提示 | P3 规划 |
